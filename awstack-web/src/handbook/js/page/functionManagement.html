功能管理

支持通过页面，安装云平台的高级扩展功能，包括物理机管理、容器管理以及腾讯PaaS服务管理。

注意：

当系统进行对接存储或者修改密码时，建议不要进行容器管理和物理机管理的功能安装，并且，不能同时进行容器管理和物理机管理的功能安装。

添加容器管理

前提条件

  • 需要至少两台物理服务器来安装kubernetes集群。

  • Master节点Node节点必须和云管平台的管理网连通。

  • 当选择使用Ceph集群时，每台服务器需要至少三个网卡，分别连接云管平台的管理网，存储网，集群网，并需要事先配置好相应网段的IP地址。

  • 服务器操作系统必须为CentOS-7.3及以上版本。

  • 获取文件 kube-kit-v1.8.tar.gz并放置在物理节点相应路径（如/opt）下，执行如下命令解压并上传镜像到docker images。 gzip -dc kube-kit-v1.8.tar.gz |docker load

操作步骤

  1. 在云管平台的Web控制台上，选择“产品和服务 >系统管理 >系统设置”，进入系统设置导航菜单。

  2. 在左侧导航菜单中单击“功能管理”，进入功能管理页面。

  3. 单击“添加”按钮。

  4. 在下拉列表中选择“容器管理”，配置容器管理功能相关信息。

  5. 单击“确定”按钮。

  6. 在容器管理页签，单击“安装”按钮。 待安装完成后，操作状态为“安装成功”。在导航栏可以查看容器管理菜单。如安装失败，单击“清除”按钮，然后重新安装。

  7. 单击功能名称列的“容器管理”，查看安装日志。

添加物理机管理

注意：

  1. 使用软件交付模式部署的云管平台需要对接共享存储后，才能进行物理机管理功能安装。

  2. 被纳管的物理机和宿主机集群网必须是连通的，被纳管的物理机IPMI和宿主机管理网必须网络连通。

前提条件

需要将获取到的 inspector-vm.qcow2和 ironic-vm.qcow2文件放置所有物理节点的“/var/lib/awstack/”路径下。

操作步骤

  1. 在云管平台的Web控制台上，选择“产品和服务 >系统管理 >系统设置”，进入系统设置导航菜单。

  2. 在左侧导航菜单中单击“功能管理”，进入功能管理页面。

  3. 单击“添加”按钮。

  4. 在下拉列表中选择“物理机管理”，配置物理机业务网络相关信息。

  5. 单击“确定”按钮。

  6. 在物理机管理页签，单击“安装”按钮。 待安装完成后，操作状态为“安装成功”。如安装失败，单击“清除”按钮，然后重新安装。

  7. 单击功能名称列的“物理机管理”，查看安装日志。

管理PaaS服务

云管平台通过功能管理模块控制云管平台启用哪些PaaS服务，只有在功能管理中添加的PaaS服务才能在云管平台页面中显示。其中功能管理中可添加功能项应受到Licence控制。

添加PaaS服务

前提条件

各PaaS服务已在物理服务器或云主机上部署完成。部署方法请参考服务提供商提供的相关文档。

操作步骤

  1. 在云管平台的Web控制台上，选择“系统管理系统设置”，进入系统设置导航菜单。

  2. 在左侧导航菜单中单击“功能管理”，进入功能管理页面。

  3. 单击“添加”按钮。

  4. 在下拉列表中选择对应的“PaaS服务”，PaaS服务可选项有：TDSQL、CTSDB、TBase、TBDS、云镜、织云、灵雀。配置PaaS服务相关信息。

  5. 单击“确定”按钮。

当在功能管理中，添加成功PaaS服务后，会在顶部导航页中，看到对应PaaS服务的入口信息。

清除服务

若PaaS服务需要从云平台中移除，可通过“清除”按钮，来将PaaS服务移除。

织云同步数据

当织云服务安装完成后，系统会自动将云主机、物理机等信息导入织云的CMDB。同时，也可通过“同步数据”操作将云管平台的云主机、物理机等信息手动同步至织云的CMDB。
