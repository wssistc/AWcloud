安全组

安全组是一个逻辑上的分组，可以实现相互信任的云主机之间的通信。可以将同一地域内具有相同网络安全隔离需求的云主机加到同一个安全组内，从而这些云主机使用相同的安全策略。

安全组作用范围是在云主机上，更具体来说是作用在云主机的端口而不是网络上。安全组和防火墙的区别：

  • 防火墙：路由级别做隔离
  • 安全组：云主机级别做隔离

可以通过安全组策略对云主机的出入流量进行安全过滤，封闭以及开启特定的端口，以保证云主机的安全访问。默认的，只有22端口（使用ssh）和3389（使用rdp）的端口是开放的，其他端口默认都是关闭的。

新建安全组

  1. 在安全组列表页，单击页面右上方的“新建”按钮。
  2. 输入安全组名称、描述，单击“下一步”。
  3. 新建安全组规则。

            安全组规则的参数设置介绍如下：

  • TCP、UDP规则：过滤基于TCP、UDP规则的端口号。
    方向：入口和出口。
    端口：可指定端口和端口范围。如果指定端口，则需要在下面填入端口号（范围1～65535）；如果指定端口范围，则填入起始端口和结束端口。
    方式：CIDR，指定网络网段。安全组，指定某一安全组。如：允许从网段10.5.0.0/24，端口范围34-56的TCP访问。
  • ICMP规则：过滤ICMP规则。
    方向：入口和出口。
    类型和编码：ICMP报文中的类型字段，和编码一起使用。如类型字段8，编码字段0，代表ping请求回显；类型字段0，编码字段0，代表ping请求应答。不指定则填-1。
    方式：CIDR，指定网络网段。安全组，指定某一安全组。如：允许0.0.0.0/0（所有）的ICMP访问。
  • ALL ICMP规则：过滤所有的ICMP规则。
    方向：入口和出口。
    方式：CIDR，指定网络网段；安全组，指定某一安全组。具体含义同上。

         4. 单击“完成”按钮。操作成功后，可以在安全组列表区查看。

删除安全组

注意：
当安全组内有一个或多个云主机，需要将这些云主机与其他安全组关联，与该安全组解除关联。该安全组才能被删除。

删除安全组，安全组规则也将删除。

  1. 在安全组列表页，勾选待删除的安全组，单击“删除”按钮。
  2. 在弹出的确认对话框单击“确定”按钮。

检查安全组规则是否生效

通过以下方法，检测前面添加的安全组规则是否生效。

  1. 前面云主机加载的安全组firewall_test中，有一条默认规则是允许云主机访问外部网络。那么执行ping命令，可以ping通外部网络。
  2. 从另外一台云主机中，启动控制台，访问刚才新建的云主机，发现无法访问，是由于安全组中没有这条规则，外部目标无法访问。
  3. 添加外部网络可以访问的规则。在云管平台的Web控制台上，单击左侧导航栏的“虚拟资源管理”，然后在页面右侧的项目列表区点击项目进入项目的概况页面。然后单击“安全”页签并选择“安全组”。单击安全组名称“firewall_test”。添加规则。
  4. 添加规则之后，可以看到firewall_test中多了一条规则，允许从目标default的ICMP访问。
  5. 再从另外一台主机来访问，发现可以ping通这台主机了。
