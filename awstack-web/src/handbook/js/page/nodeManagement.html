节点管理

加入集群

说明：
新增节点注册成功后才能进行初始化配置后加入集群。注册过程请参见《AWCloud云计算管理平台 安装部署手册》。

注意：
选择“系统设置> 软件授权”查看当前软件的授权信息，当“支持节点数”为“N/A”时，在“节点管理”页面添加节点数将不受限制。当“支持节点数”为具体的数值时，则“节点管理”页面已使用的虚拟资源节点数必须小于“支持节点数”时，才能进行新增节点操作。

  1. 在云管平台的Web控制台上，选择“产品和服务>系统管理>运维管理”，进入运维管理导航菜单。
  2. 在左侧导航菜单中单击“节点管理”，进入节点管理页面。
  3. 在节点管理页面，选择“虚拟资源节点”页签，新节点注册成功后，显示状态为“未配置”，单击“加入集群”按钮，对节点进行网卡、网络、硬盘配置。
  4. 依次配置完成后，在配置页面单击“加入集群”按钮，系统状态变为“开始新增节点”，配置成功后，健康状态由“N/A”变为“健康”，节点状态变为“节点管理成功”。

节点晋升

云管平台使用两节点安装部署时，通过节点晋升功能将新增纯计算节点晋升为管理+计算的节点。并将新加入的第三个物理节点也晋升为管理+计算节点。

注意：
节点晋升功能只针对两节点晋升为三节点。节点晋升过程中两节点高可用功能将不可用，且晋升过程不可逆。

  1. 在云管平台的Web控制台上，选择“产品和服务 >系统管理 >运维管理”，进入运维管理导航菜单。
  2. 在左侧导航菜单中单击“节点管理”，进入节点管理页面。
  3. 在节点管理页面，选择“虚拟资源节点”页签，在节点列表中选择两个状态为“节点管理成功”的节点，单击“启动多节点高可用架构”按钮。
  4. 在配置页面单击“节点晋升”按钮，并在弹出的确认框内单击“确定”。
  5. 单击“下一步”按钮，在节点列表中勾选注册的新节点，单击“加入集群”按钮。
  6. 对新节点的网卡信息、网络信息和硬盘信息依次进行配置，配置完成后，单击“加入集群”按钮，节点状态变为“节点配置成功”。
  7. 选择加入集群的新节点，单击“新节点晋升”按钮，将第三个物理节点也晋升为管理+计算节点。
  8. 在弹出的确认框里单击“确定”按钮然后单击“完成”按钮。

维护虚拟资源节点

注意：
使用两个或三个节点部署时，不允许对任一节点进行维护操作。使用四个节点部署时，仅支持对其中一个节点进行维护。

当虚拟资源节点出现硬件问题时，对虚拟资源节点进行维护。维护节点前需确保当前节点的所有云主机已经迁移或清空，否则会造成数据丢失。

在“虚拟资源节点”页签列表区，勾选待操作的节点，单击“维护”按钮。在弹出的确认框里单击“确定”按钮。

激活虚拟资源节点

注意：
使用两个或三个节点部署时，不允许进行激活操作。

当虚拟资源节点从维护状态重新启用时，单击“激活”按钮，将节点重新启用。

删除虚拟资源节点

注意：
云管平台存在三个节点时，不允许进行删除操作。删除节点前需确保当前节点的所有云主机已经迁移或清空。当使用软件交付模式部署云管平台时，不允许删除node-1节点，因为镜像和glance-api都保存在node-1上。

在“虚拟资源节点”页签列表区，勾选待操作的节点，单击“删除”按钮。在弹出的确认框里单击“确定”按钮。

强制删除虚拟资源节点

当云管平台部署了多个数据中心时，由于某种原因直接重装了某个数据中心，导致数据库有残留的节点信息。通过强制删除操作将残留的节点信息删除。

在“虚拟资源节点”页签列表区，勾选待操作的节点，在“更多操作”列表选择“强制删除”按钮。在弹出的确认框里输入超级管理员admin登录信息。
